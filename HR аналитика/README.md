# HR-аналитика и прогнозирование удовлетворённости сотрудников

## О проекте
Проект представляет собой комплексное решение для HR-аналитики компании "Работа с заботой", включающее два взаимосвязанных направления:
1. **Прогнозирование уровня удовлетворённости сотрудников** - регрессионная задача для оценки настроений в коллективе
2. **Прогнозирование рисков увольнения** - классификационная задача для проактивного управления оттоком персонала

### Бизнес-контекст:
Компания предоставляет HR-аналитические услуги организациям, помогая оптимизировать управление персоналом. Проект позволяет перейти от периодических опросов к непрерывной аналитике и превентивному управлению.

## Цели проекта
1. **Аналитическая**: Исследовать факторы, влияющие на удовлетворённость и увольнение сотрудников
2. **Прогностическая**: Построить точные модели для предсказания удовлетворённости и рисков оттока
3. **Практическая**: Разработать рекомендации для улучшения HR-процессов
4. **Бизнес-ориентированная**: Снизить затраты на опросы и подбор персонала

## Ключевые результаты

### Задача 1: Прогнозирование удовлетворённости
- **Лучшая модель**: DecisionTreeRegressor
- **Качество (SMAPE)**: 13.93% на тестовой выборке
- **Превосходство над базовой моделью**: 2.7 раза (38.26% → 13.93%)
- **Ключевые факторы**: 
  - Оценка руководителя (корреляция 0.73)
  - Наличие нарушений (0.59)
  - Получение повышений (0.43)

### Задача 2: Прогнозирование увольнений
- **Лучшая модель**: DecisionTreeClassifier
- **Качество (ROC-AUC)**: 0.9154 на тестовой выборке
- **Превосходство над базовой моделью**: 0.5 → 0.9154
- **Ключевые факторы**:
  - Стаж работы (корреляция 0.66)
  - Уровень зарплаты (0.56)
  - Уровень должности (0.31)

### Дополнительные инсайты:
- **Портрет увольняющегося сотрудника**: Junior, 1 год опыта, отдел Sales/Technology, низкая зарплата
- **Бимодальность распределений**: Два пика в распределении зарплат и удовлетворённости
- **Статистическая проверка**: Удовлетворённость уволившихся значимо ниже (p-value < 1.4e-101)

## Выявленные проблемы и решения

### Проблемы:
1. **Дисбаланс данных**: 90% без повышений, 85% без нарушений, 75% не увольняются
2. **Опечатки в данных**: 'sinior' вместо 'senior'
3. **Пропуски значений**: В категориальных признаках dept и level
4. **Дубликаты**: Значительное количество в разных выборках

### Решения:
1. **Стратегии работы с дисбалансом**: Использование StratifiedKFold, метрики ROC-AUC
2. **Автоматическая коррекция**: Пайплайны для исправления опечаток
3. **Заполнение пропусков**: SimpleImputer с strategy='most_frequent'
4. **Очистка дубликатов**: Последовательная обработка всех выборок

## Бизнес-воздействие

- **Оптимизация HR-бюджета**: Сокращение расходов на опросы на 40-50%
